# TripTogether (TripPlanner)

Веб-сервис для совместного планирования групповых поездок: поездки, участники, голосования, бюджет и расчёт долгов.

## Стек

- **Backend:** FastAPI, PostgreSQL 15
- **Инфраструктура:** Docker Compose

## Сервисы

| Сервис          | Порт | Назначение                          |
|-----------------|------|-------------------------------------|
| auth-service    | 8001 | Регистрация, вход, JWT, профиль     |
| user-service    | 8002 | Профили пользователей (данные из Auth) |
| trip-service    | 8003 | Поездки, участники, приглашения     |
| voting-service  | 8004 | Опросы и голосования по поездке      |
| budget-service  | 8005 | Расходы и расчёт долгов             |
| chat-service         | 8006 | Чат по поездке (сообщения)          |
| notification-service  | 8007 | Email-уведомления (SMTP)            |
| PostgreSQL           | 5432 | БД                                  |
| pgAdmin              | 5050 | Админка БД                          |

## Что сделано

- **Auth:** регистрация, логин, верификация токена, `GET/PATCH /auth/me` (профиль, имя, avatar_url)
- **Поездки:** создание, список, просмотр, удаление; доступ по создателю и участникам
- **Участники:** приглашение по email, список участников, принятие приглашения
- **Голосования:** создание опроса, варианты ответов, голос (один на пользователя), результаты
- **Бюджет:** добавление расхода (сумма, категория, между кем делим), список расходов, расчёт долгов «кто кому сколько должен»
- **Чат:** отправка сообщения (REST и WebSocket), список сообщений (пагинация). WebSocket — доставка в реальном времени
- **Профиль:** PATCH /auth/me (имя, avatar_url), GET /profiles/me и /profiles/{id} — данные из Auth
- **Уведомления:** notification-service (SMTP); письма при приглашении в поездку, новом голосовании, новом сообщении в чате

## Запуск

```bash
docker-compose up -d
```

## Frontend

В каталоге `frontend/` — SPA на **React + Vite + TypeScript**. Запуск: `cd frontend && npm install && npm run dev` (порт 5173).

## Тесты

Тесты лежат в каталоге `tests/` по сервисам: `trip_service`, `voting_service`, `budget_service`.
